## 🔴 핵심 인시던트 상세 분석

### 인시던트 1: Gemini API 갑자기 작동 중단 (가장 중요)

**발견 시점**: 4/9 16:20경 (수업 시작 직후)
**증상**: 모든 AI 도구에서 "예시 데이터" 표시, AI 결과 안 나옴
**에러 메시지**: `403 - Your project has been denied access. Please contact support.`

#### 왜 갑자기 일어났나? (핵심 분석)

**배경**:
- 3/9에 `gemini-2.0-flash` → `gemini-2.5-flash`로 업그레이드 (커밋 befed79)
- 이유: Google이 2.0-flash의 free tier를 limit:0으로 차단했기 때문
- 3/9 ~ 4/9 3시까지 약 1달간 `gemini-2.5-flash`로 **정상 작동**

**왜 4/9에 갑자기 안 됐는가?**

이건 **우리 코드 문제가 아니라 Google 쪽 변경**이었습니다:

1. **Google의 `gemini-2.5-flash` free tier 정책 변경 (가장 유력)**
   - Google은 Gemini 모델의 free tier 접근을 수시로 조정합니다
   - 3/9에 작동하던 2.5-flash가 4/9에 갑자기 `503 UNAVAILABLE (high demand)` 반환
   - 이는 Google이 해당 모델의 무료 사용자 접근을 제한했다는 의미
   - 우리가 아무런 코드 변경 없이도 Google 정책 변경만으로 갑자기 안 될 수 있음

2. **API 키의 Google Cloud 프로젝트 자동 차단 (보조 원인)**
   - 첫 에러는 `403 project denied access` — 프로젝트 레벨 차단
   - Google이 해당 프로젝트를 남용 감지(abuse detection)로 자동 차단했을 가능성
   - 교육 목적이라도 짧은 시간에 대량 호출하면 자동 플래그 가능

3. **모델별 free tier 현황 (4/9 실측 결과)**

   | 모델 | 상태 | 설명 |
   |---|---|---|
   | gemini-2.5-flash | 503 | 서버 과부하 / free tier 제한 |
   | gemini-1.5-flash | 404 | v1beta API에서 완전 제거됨 |
   | gemini-2.0-flash | 429 | quota 소진 (이미 limit:0) |
   | **gemini-2.5-flash-lite** | ✅ | **유일하게 작동** |

4. **결론**
   - "개발을 안 했는데 왜 갑자기?" → **Google이 서버 쪽에서 모델 접근 정책을 바꿨기 때문**
   - 이건 외부 의존성(Google API) 사용 시 피할 수 없는 리스크
   - 해결: `gemini-2.5-flash-lite`로 교체 (4/9 17:02, 커밋 82be4f8)

#### 예방 방안
- **모델 fallback 시스템**: 기본 모델 실패 시 자동으로 다른 모델 시도
- **유료 플랜 고려**: free tier는 Google이 언제든 정책 변경 가능
- **모델 health check**: 앱 시작 시 API 1회 ping으로 모델 상태 확인

### 인시던트 2: 카카오/구글 소셜 로그인 안 됨

**발견 시점**: 4/9 16:20
**증상**: 카카오/구글 버튼 클릭 시 `400 - Unsupported provider: provider is not enabled`
**원인**: Supabase 프로젝트에 **카카오/구글 OAuth가 enable되어 있지 않았음** (client_id/secret 미설정)
**처리**: 소셜 로그인 버튼 숨김 (`VITE_ENABLE_SOCIAL_LOGIN` 플래그)
**영향**: 이메일 가입으로 전환 — 외국인 학생은 어차피 카카오 계정 없어서 실질 영향 없음

#### 미조치 사항
- ⚠️ 카카오/구글 OAuth 앱 발급 → Supabase에 client_id/secret 설정 필요
- 설정 후 Vercel env에 `VITE_ENABLE_SOCIAL_LOGIN=true` 추가하면 버튼 복원

### 인시던트 3: 빈 화면 (아이보리 + 상단바만) 반복

**발견 시점**: 4/9~10 수업 내내 간헐적
**증상**: 페이지 이동 시 콘텐츠 안 뜸 → 새로고침 필요
**근본 원인 (2가지)**:
1. `PageTransition` 컴포넌트의 `animation-fill-mode: both`가 `opacity: 0`에 고정 → 빈 화면
2. PWA Service Worker가 옛날 번들을 캐시 → 새 코드 안 먹음

**처리**:
1. `animation-fill-mode: both` → `forwards`로 변경
2. `PageTransition`을 `key={pathname}` 기반 리마운트로 변경
3. `index.html`에 SW 강제 해제 스크립트 추가 (모든 SW unregister + 캐시 삭제)
4. 스크롤/카드 투명도 애니메이션 전부 비활성화

**효과**: SW 해제 후 빈 화면 발생 대폭 감소

## 💡 교훈 & 권장 사항

### 1. 외부 API 의존성 관리
> Google Gemini free tier는 **공지 없이** 모델 접근을 차단할 수 있다.
> → 모델 fallback + health check + 유료 플랜 검토 필수

### 2. 캐시 관리
> PWA Service Worker 캐시가 빠른 배포 사이클에서 **가장 큰 적**.
> → 개발/안정화 단계에서는 SW 비활성화 권장

### 3. RLS 정책 사전 검증
> "강사만 INSERT"인 테이블에 학생이 가입 시 write → 조용히 실패.
> → 가입 플로우의 모든 DB 쓰기에 대해 RLS 통과 여부 사전 테스트 필수

### 4. 현장 대응 체계
> 수업 중 실시간 버그 수정 + 배포는 가능하지만 **TS 에러 하나가 전체 배포를 막을 수 있음**.
> → 빌드 전 `tsc --noEmit` 자동 실행 + TS strict mode 적용 검토
