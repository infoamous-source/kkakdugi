교안 슬라이드에 반영해야 할 변경 사항을 정리합니다.

mockup 시각 참고: `docs/mockups/index.html` (브라우저로 열어 AFTER 컬럼 확인)

> 📌 작업 진행 상황: 교안에 영향이 큰 도구부터 순차적으로 갱신됩니다.
> 각 교시 섹션이 추가될 때마다 교안 담당자에게 알림이 갑니다.

## 3교시 — 엣지메이커 (Edge Maker) — ✅ 완료 (2026-04-07)

### 한 줄 요약
**자동 로드 → 수동 로드**로 바뀌었고, 강점 입력이 **PlusListInput**으로 통일됐어요.
보석함 버튼이 통일된 검정 버튼으로 바뀌었고, 다음 교시 CTA가 추가됐어요.

### 학생이 보게 될 화면
- mockup: `docs/mockups/edge-maker.html` (AFTER 컬럼)
- 실제 코드: `src/pages/marketing/school/tools/EdgeMakerTool.tsx`

### 주요 변경
1. **2교시 데이터 자동 로드 → 수동 버튼**: `📥 2교시 마켓스캐너 데이터 불러오기` 누를 때만 가져옴
2. **나의 강점 입력**: 기존 `<input> + [+] 버튼 + 태그 목록` → **PlusListInput** (1️⃣ ✖ ➕ 패턴)

4. **보석함 저장**: "💎 보석함에 넣기" → **"결과를 아이디어보석함에 저장하기 →"** (검정 큰 버튼)

6. **"no scanner data" 경고 화면 제거** — 이제 수동 로드 버튼만 보이면 됨

### 기존 교안에서 바꿔야 할 것

- ✏️ **수정**: 강점 입력 설명 → PlusListInput 패턴으로 통일
- ➕ **추가**: "데이터 불러오기" 버튼 개념 — 학생이 원할 때만 이전 결과를 가져오는 흐름

### 슬라이드에 들어갈 안내 카피
- 입력 단계: "2교시 마켓스캐너 결과를 '불러오기 📥' 버튼으로 가져올 수 있어요. 누를 때만 가져오고, 누르지 않으면 강점만 직접 입력해도 돼요."

## 4교시 — 바이럴카드메이커 (Viral Card Maker) — ✅ 완료 (2026-04-07)

### 한 줄 요약
**Pexels 무료 이미지 + 5가지 레이아웃 템플릿** 위에 텍스트를 얹어서 인스타그램에 바로 업로드할 수 있는 카드 4장을 만들어요.

### 학생이 보게 될 화면
- mockup: `docs/mockups/viral-card.html` (AFTER 컬럼)
- 실제 코드: `src/pages/marketing/school/tools/ViralCardMakerTool.tsx`
- 템플릿 컴포넌트: `src/pages/marketing/school/tools/common/ViralCardTemplates.tsx`

### 입력 흐름
1. 📥 3교시 엣지메이커 데이터 불러오기 (선택) — 상품명·USP 자동 채움
2. 상품명 (예: DripQ 캡슐)
3. **타겟 페르소나** — +버튼으로 여러 명 추가 (예: 30대 직장인 여성 / 출퇴근족)
4. USP (차별점)
5. **이미지 배경** — 📷 실사 무료이미지(Pexels) / 🎨 AI 생성 (AI는 추후 지원)
6. **분위기 선택** — 🌶️ 자극형 / 💗 감성형 / 📊 정보형
7. 「4장 카드뉴스 생성 →」 클릭

### 결과 화면 구성
4장이 **2x2 그리드**로 렌더링돼요. 각 카드는 **mockup의 5가지 레이아웃 템플릿(A~E) 중 하나** 사용:
- **카드 1 (Template A)** — 상단 흰 박스 + 하단 이미지 — "HOOK · 시선잡기"
- **카드 2 (Template B)** — 풀이미지 + 하단 검은 그라데이션 — "EMPATHY · 공감"
- **카드 3 (Template C)** — 풀이미지 + 가운데 큰 글씨 + **브랜드명 노출** — "SOLUTION · 해결"
- **카드 4 (Template D)** — 좌측 주황 컬러박스 + 우측 이미지 + **브랜드명 노출** — "ACTION · 행동"

각 카드:
- 좌상단 **단계 배지** (주황, 예: "1 · 시선잡기")
- **AI가 생성한 짧은 카피** (2-3줄, 특정 단어는 노란색 하이라이트)
- **Pexels 이미지** (실시간 검색, 30명 동시 사용 시 sessionStorage 캐시)
- 카드 3·4번은 **브랜드명이 노란 배지로 카드에 노출** (인스타에서 브랜드 인지)
- 우하단 **📷 Pexels** 출처 표기

하단:
- 🔁 **다른 이미지로** (작은 회색 텍스트 링크) — 같은 키워드로 이미지만 랜덤 교체
- 📥 **PNG 받기** (추후 지원 예정)
- **결과를 아이디어보석함에 저장하기 →** (검정 큰 버튼)

### 슬라이드에 들어갈 안내 카피
- 입력 단계: "이번에는 인스타그램 카드뉴스 4장을 만들어볼게요. 4장이 하나의 스토리예요: 시선잡기 → 공감 → 해결 → 행동 순서로."
- 타겟 페르소나: "한 명만 생각하지 말고 여러 명을 상상해보세요. 각자 어떤 고민을 가지고 있을까요?"
- 톤 선택: "자극형은 반말로 충격, 감성형은 존댓말로 따뜻함, 정보형은 숫자와 데이터예요. 상품에 맞게 골라주세요."
- 결과: "이 4장을 그대로 인스타그램에 업로드할 수 있어요. 이미지는 Pexels에서 자동으로 찾아와요."

### 기존 교안에서 바꿔야 할 것
- ❌ **삭제**: "spicy" 영어 용어 → "🌶️ 자극형"
- ❌ **삭제**: AI 이미지 생성(Gemini image API) 중심 설명 → Pexels 무료 이미지 중심으로
- ❌ **삭제**: 이미지 스타일 5종(illustration/realistic/minimal/popart/custom) → 단순 **📷 실사 / 🎨 AI(비활성)** 2개로
- ❌ **삭제**: imagePrompt 영어 프롬프트 개념 → imageKeyword 짧은 영어 키워드로
- ❌ **삭제**: AIDA·학습카드·designTip·colorScheme 같은 이론 설명
- ❌ **삭제**: `overallStrategy` 20줄 레포트 결과
- ✏️ **수정**: "바이럴 카드 만들기" → "바이럴 카드 만들기 🔥" (이모지 추가)
- ✏️ **수정**: 타겟 페르소나 1개 입력 → **여러 명 +버튼 리스트**
- ✏️ **수정**: "생성하기" → "4장 카드뉴스 생성 →"
- ➕ **추가**: 단계 배지 읽는 법 (1 시선잡기 / 2 공감 / 3 해결 · 브랜드ON / 4 행동 · 브랜드ON)
- ➕ **추가**: 브랜드명이 카드 3·4에만 나오는 이유 — "앞에서 감정을 잡고, 뒤에서 브랜드를 각인"
- ➕ **추가**: 5가지 레이아웃 템플릿 소개 (A: 상단박스 / B: 그라데이션 / C: 가운데큰글씨 / D: 좌우분할)

### 핵심 학습 포인트
1. **4장 공식** — 시선잡기 → 공감 → 해결 → 행동 (= AIDA 변형)
2. **Pexels 무료이미지 활용법** — 영어 키워드로 검색하는 습관
3. **텍스트 오버레이 디자인** — 이미지 위 글씨가 읽히려면 그림자·어두운 오버레이 필요
4. **브랜드 노출 타이밍** — 처음부터 노출하지 말고 관심 끈 뒤 공개
5. **톤 매칭** — 상품·타겟에 맞는 톤 선택 (MZ세대 → 자극형, 30-40대 → 감성형, 전문가 → 정보형)

### 외국인 학생용 영어 안내 (P0-4에서 일괄)
- 입력: "Enter your product, target personas (add multiple with +button), USP, and pick a tone. Image source is Pexels free photos."
- 결과: "4 cards in 2x2 grid. Each has a different layout: top-box, gradient, center-big, split. Cards 3 & 4 show your brand name."

### 개발자 노트
- `src/services/pexelsService.ts` 신규 생성: `searchPexelsImage(keyword)` / `searchPexelsImages(keywords[])` + sessionStorage 캐싱
- 환경변수 `VITE_PEXELS_API_KEY` 필요 (없으면 fallback 그라데이션)
- `ViralCardResult` 타입 전면 재설계: `imagePrompt/imageStyle` → `imageKeyword/imageSource`
- 각 slide에 `template: 'A'|'B'|'C'|'D'|'E'` + `showBrand: boolean` + `highlightWord?: string` 추가

## 6교시 — ROAS 자가 진단기 (ROAS Simulator) — ✅ 완료 (2026-04-06)

### 한 줄 요약
**시뮬레이터가 아니라 자가 진단기로 바뀌었어요.**
AI가 매출을 예측하는 게 아니라, **학생이 본인 광고비·매출을 직접 입력**하고
신호등으로 진단받는 방식이에요.

### 학생이 보게 될 화면
- mockup: `docs/mockups/roas-simulator.html` (AFTER 컬럼)
- 실제 코드: `src/pages/marketing/school/tools/ROASSimulatorTool.tsx`

### 입력 흐름 (학생이 따라하는 순서)
1. 광고에 얼마 썼어요? (예: 300,000원)
2. 매출이 얼마였어요? (예: 540,000원)
3. 어디에 광고했어요? (인스타/네이버/카카오/유튜브 중 1개)
4. 「확인하기 →」 클릭

### 결과 화면 구성 (위에서 아래로)
1. **큰 ROAS 숫자 (54px)** — 예: `1.8×`
2. **한 줄 의미** — "광고비 1원 → 매출 1.8원"
3. **신호등** — 🔴 손해 / 😐 본전 / 📈 좋아요 (3개 중 1개만 컬러로 빛남)
4. **큰 한 줄 처방** — 예: "👇 광고비를 줄이세요 / 30만원 → 20만원으로"
5. **오늘 할 일 1개** (노란 카드) — 예: "광고 사진을 새 걸로 바꿔보세요"
6. **자세한 숫자 4개** — 광고비 / 매출 / 남은 돈(이익) / ROAS
7. **계산식 한 줄** — "매출 540,000 ÷ 광고비 300,000 = 1.8"
8. **다시 해보기** (작은 회색 텍스트 링크) — 폼으로 돌아가서 숫자 바꿔보기
9. **결과를 아이디어보석함에 저장하기 →** (검정 큰 버튼)
10. **🎓 졸업하기 · 최종 과제 제출 →** (졸업 CTA)

### 신호등 기준
- **ROAS < 1.5** → 🔴 손해
- **1.5 ≤ ROAS ≤ 2.5** → 😐 본전
- **ROAS > 2.5** → 📈 좋아요

### 슬라이드에 들어갈 안내 카피 (이 문구로 안내해주세요)
- 입력 단계: "여러분이 직접 운영해본(또는 운영할 예정인) 광고의 광고비와 매출을 적어주세요. AI가 추정하는 게 아니에요. 본인이 진짜 숫자로 진단받는 거예요."
- 결과 단계: "ROAS가 2.0이면 본전, 2.0보다 낮으면 손해, 높으면 이익이에요. 신호등이 빨간색이면 광고비를 줄이거나 광고 소재를 바꿔야 해요."
- 다시 해보기: "광고비를 절반으로 줄이면 신호등이 어떻게 변할지 직접 바꿔보세요. 정해진 답은 없어요."

### 기존 교안에서 바꿔야 할 것
- ❌ **삭제**: 노출(impressions), 클릭(clicks), CTR, CVR, CPC, CPA 같은 전문 지표 설명
- ❌ **삭제**: 펀넬 차트(노출→클릭→전환) 시각화
- ❌ **삭제**: AI가 시뮬레이션한다는 표현
- ❌ **삭제**: 상품명/상품가격/광고기간/타겟 연령 입력
- ❌ **삭제**: roasGrade의 4단계(excellent/good/average/poor) → 3단계 신호등으로
- ✏️ **수정**: "ROAS 시뮬레이터" → "내 광고 점수 확인하기 💰"
- ✏️ **수정**: "시뮬레이션 시작" → "확인하기 →"
- ✏️ **수정**: 입력 항목이 5개 → **3개로 단순화** (광고비, 매출, 채널)
- ➕ **추가**: ROAS = 매출 ÷ 광고비 공식을 슬라이드 1장으로 따로 설명
- ➕ **추가**: 신호등 3단계 (손해/본전/좋아요) 의미 설명 슬라이드
- ➕ **추가**: "본전(2.0)" 개념 강조 — 광고비 1원 = 매출 2원이 손익분기

### 핵심 학습 포인트 (교안 강조해주세요)
1. **ROAS 개념을 수식으로 직접 체험** — "매출 ÷ 광고비"
2. **본전(2.0) = 손익분기점** — 광고비 외 원가까지 고려한 실전 기준
3. **신호등 = 직관적 진단** — 숫자로 헷갈리지 말고 색으로 판단
4. **다시 해보기** — 광고비/매출 바꿔가며 ROAS 변화를 몸으로 익히기
5. **외국인 학생 친화** — 노출/CTR 같은 영어 약어 없이 모국어로 설명 가능

### 외국인 학생용 영어 안내 (영어 토글 시 — P0-4에서 일괄 작업 예정)
- 입력 단계: "Enter the actual ad spend and revenue you got from your ad campaign."
- 결과 단계: "ROAS = revenue ÷ ad spend. If ROAS < 1.5, you're losing money. 1.5–2.5 is breakeven. Above 2.5 means profit."
- 다시 해보기: "Try changing the numbers to see how the signal light changes. There's no right answer."

### 개발자 노트
- ROAS 계산은 클라이언트에서 즉시 수행 (`calculateRoas()` 함수). AI 호출 0회.
- AI 호출은 **한 줄 처방 + 오늘 할 일 1개**를 받기 위한 가벼운 호출 1회만. 30명 동시 부하 안전.
