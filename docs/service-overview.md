# 깍두기학교 서비스 소개서

> 이주민/유학생을 위한 실무 교육 플랫폼

---

## 1. 서비스 개요

깍두기학교는 한국에 거주하는 이주민과 유학생을 위한 **웹 기반 교육 플랫폼**입니다. 디지털 기초, 마케팅 실무, 커리어 개발의 3개 학과를 운영하며, AI 실습 도구와 게이미피케이션을 결합한 학습 경험을 제공합니다.

| 항목 | 내용 |
|------|------|
| 플랫폼 | 모바일 우선 반응형 웹앱 (PWA) |
| 학과 수 | 3개 (디지털 / 마케팅 / 커리어) |
| 전체 모듈 | 15개 |
| 전체 수업 | 96개 |
| AI 실습 도구 | 14개 (학교 6개 + 프로 8개) |
| 지원 언어 | 한국어, 영어 |
| 기술 스택 | React 19, TypeScript, Tailwind CSS v4, Vite 7, Supabase, Gemini AI |

## 5. AI 실습 도구 전체 상세

### 5-1. 예비 마케터 교실 - 6교시 AI 도구

교시 간 데이터가 순차적으로 연계되어 하나의 마케팅 프로젝트를 완성합니다.

#### 1교시: 직무 적성 테스트 (Marketer DNA Test)
| 항목 | 내용 |
|------|------|
| 라우트 | `/marketing/school/tools/aptitude-test` |
| 기능 | 마케터 유형 진단 (5개 페르소나) |
| 입력 | 객관식 질문 응답 |
| 출력 | 페르소나 결과 (CEO/PM/CPO/CMO/CSL), 점수 분석 |
| 데이터 연계 | 학생증에 페르소나 배지 표시 |

#### 2교시: 시장 스캐너 (Market Scanner AI)
| 항목 | 내용 |
|------|------|
| 라우트 | `/marketing/school/tools/market-scanner` |
| AI | Gemini API (generateMarketAnalysis) |
| 기능 | 키워드 기반 시장 분석 |
| 입력 | 상품 키워드, 타겟 연령, 성별, 상품 유형 |
| 출력 | 연관 검색어 5개, 경쟁사 분석(강점/약점), 고객 페인포인트 |
| 데이터 연계 | 3교시(브랜드 빌더)에 pain points, competitors 자동 전달 |

#### 3교시: 브랜드 빌더 (Brand Builder AI)
| 항목 | 내용 |
|------|------|
| 라우트 | `/marketing/school/tools/edge-maker` |
| AI | Gemini API (generateBrandingStrategy) |
| 기능 | 브랜드 전략 수립 |
| 입력 | 2교시 데이터 자동 로드 + 나의 강점 (최대 5개) |
| 출력 | USP 문장, 브랜드 네이밍 3종(감정형/직관형/재미형), 슬로건, 브랜드 무드(색상/톤/키워드) |
| 데이터 연계 | 4교시, 5교시에 brand name, USP 자동 전달 |

#### 4교시: 콘텐츠 메이커 (Content Maker AI)
| 항목 | 내용 |
|------|------|
| 라우트 | `/marketing/school/tools/viral-card-maker` |
| AI | Gemini API (generateViralCards, generateAllSlideImages) |
| 기능 | 4단계 바이럴 카드 제작 (Hook → Empathy → Solution → Action) |
| 입력 | 톤 선택(Spicy/Emotional/Informative), 이미지 스타일(5종), 3교시 데이터 자동 로드 |
| 출력 | 슬라이드별 카피텍스트, 이미지, 색상 스킴 |

#### 5교시: 세일즈 라이터 (Sales Writer AI)
| 항목 | 내용 |
|------|------|
| 라우트 | `/marketing/school/tools/perfect-planner` |
| AI | Gemini API (generateSalesPlan) |
| 기능 | 판매 전략 3종 생성 |
| 입력 | 제품명, 핵심 타겟, 강력한 오퍼 (3교시 데이터 자동 로드) |
| 출력 | 랜딩페이지 계획(헤드라인/문제/기능/신뢰신호/CTA), 라이브커머스 스크립트(오프닝/데모/Q&A/클로징), 영업 로직 |

#### 6교시: ROAS 예측기 (ROAS Forecast AI)

## 6. 기관용 관리 기능

### 6-1. 기관 코드 시스템

| 항목 | 내용 |
|------|------|
| 코드 형식 | 3글자 + 3숫자 (예: ABC234) |
| 자동 생성 | 헷갈리는 문자(I,L,O)와 숫자(0,1) 제외 |
| 충돌 처리 | 중복 시 최대 3회 재시도 |
| 강사 연결 | 기관 생성 시 강사 ID 자동 연결 |

### 6-2. 학생 상태 관리

```
학교 연결 → [pending_info] → 추가 정보 입력 → [active] → 졸업 → [completed]
                                                  ↓
                                          관리자 해제 → [suspended] → 재연결 → [active]
```

| 상태 | 설명 | 표시 |
|------|------|------|
| pending_info | 정보 수집 대기 | 주황색 경고 |
| active | 수강 중 | 녹색 활성 |
| suspended | 일시정지 | 회색 비활성 |
| completed | 수료 | 졸업 완료 |

### 6-3. 다중 기관 동시 지원

- 각 기관은 독립적인 코드와 강사 ID 보유
- 학생 프로필에 `org_code` 저장
- 기관별 학생 수 독립 조회
- 관리자 대시보드에서 기관별 학생 목록 관리

### 6-4. 관리자 대시보드 기능

| 기능 | 설명 |
|------|------|
| 기관 관리 | 기관 생성/조회, 코드 복사, 학생 수 확인 |
| 학생 배정 | 학생 검색, 학교 연결/해제, 상태 전환 |
| 팀 관리 | 교실/팀 편성 |
| AI 사용 현황 | 학생별 도구 사용 횟수, 포트폴리오 통계 |
| 콘텐츠 관리 | 학습 콘텐츠 편집 |
| 공지사항 | 알림/공지 발송 |

## 9. 게이미피케이션 요소

### 9-1. 학생증

학생증은 등교 탭에서 확인 가능하며, 다음 정보를 포함합니다.

| 요소 | 내용 |
|------|------|
| 학교명 | K-마케터 예비교실 |
| 이름 | 사용자 이름 (첫글자 아바타) |
| 이메일 | 가입 이메일 |
| 페르소나 배지 | 1교시 적성검사 결과 (CEO/PM/CPO/CMO/CSL) |
| 소속 | 기관명 |
| 담임 | 강사 이름 |
| 교실 | 배정된 학과명 |
| 학번 | 자동 생성 ID |
| 상태 | 재학 중 / 졸업 |
| 졸업 배지 | 졸업 시 표시 |

**디자인:** 크림→피치 그라디언트 배경, 따뜻한 톤, 브라운 테두리

### 9-2. 출석 도장판

| 요소 | 내용 |
|------|------|
| 칸 수 | 6칸 (6교시) |
| 레이아웃 | 2-3열 반응형 그리드, 정사각형 |
| 완료 상태 | 체크마크 + 컬러 그라디언트 |
| 미완료 상태 | 자물쇠 아이콘 + 회색 배경 |
| 진행 바 | purple → pink 그라디언트, 완료율 % 표시 |

**6칸 색상:**
1. rose (1교시: 기초 다지기)
2. blue (2교시: 시장 조사)
3. amber (3교시: 브랜드 기획)
4. purple (4교시: 광고 만들기)
5. emerald (5교시: 설득의 기술)
6. orange (6교시: 성과 측정)

### 9-3. 졸업 시 UI 변화

**졸업식 애니메이션 (2단계):**

**1단계 - 졸업식 (5초)**
- 화면: 자주→핑크 그라디언트 배경
- 깍두기 마스콧 축하 포즈 (양팔 들기)
- 졸업모자 날아오는 애니메이션
- Confetti 폭발 4회 (500ms, 800ms, 1500ms, 3000ms)
- 5초 후 자동 전환

**2단계 - 결과 화면**
- 프로 마케터 교실 잠금 해제 안내
- 졸업장(수료증) 받기 버튼
- 선생님께 한마디 입력란 (500자)

**수료증 포함 정보:**
- 학생 이름
- 기관명
- 과정명 (예비 마케터 양성과정, 6차시)
- 6교시 커리큘럼 목록
- 소속 팀
- 발급일자
- 도장 이미지
- 인증번호 (SGC-2026-MK-XXXX)
- PNG/PDF 다운로드 지원

## 10. 학생 결과물 데이터화

### 10-1. 저장 구조

학생이 AI 도구로 생성한 모든 결과물은 `portfolio_entries` 테이블에 JSONB 형태로 저장됩니다.

```
portfolio_entries 테이블
├── id (UUID)
├── user_id (FK: auth.users)
├── tool_id (도구 식별자)
├── module_id (모듈 식별자)
├── tool_name (도구 이름)
├── input (JSONB - 사용자 입력값)
├── output (JSONB - AI 생성 결과)
├── is_mock_data (Boolean - 목데이터 여부)
└── created_at (생성일시)
```

### 10-2. 교시별 저장 데이터 예시

| 교시 | 도구 | input 예시 | output 예시 |
|------|------|-----------|------------|
| 1교시 | aptitude-test | {answers: {q1:'A', q2:'B'...}} | {resultType:'CEO', scores:{...}} |
| 2교시 | market-scanner | {itemKeyword:'여성지갑', targetAge:'20-30'} | {relatedKeywords:[], competitors:[], painPoints:[]} |
| 3교시 | edge-maker | {painPoints:[], myStrengths:[]} | {usp:'...', brandNames:[], slogan:'...', brandMood:{}} |
| 4교시 | viral-card-maker | {productName, tone, imageStyle} | {slides:[{copyText, imageBase64, colorScheme}]} |
| 5교시 | perfect-planner | {productName, coreTarget} | {landingPage:{}, liveCommerce:{}, salesLogic:{}} |
| 6교시 | roas-simulator | {productPrice, adBudget, duration} | {estimatedROAS, advice, analysisReport} |

### 10-3. 기관 전달 가능 데이터

관리자 대시보드(`StudentAIUsageView`)에서 확인 가능:

| 데이터 | 설명 |
|--------|------|
| 도구별 사용 횟수 | 각 AI 도구를 몇 번 사용했는지 |
| AI 생성 횟수 | 실제 AI를 호출한 횟수 |
| 포트폴리오 항목 수 | 저장된 총 결과물 수 |
| 최근 활동일 | 마지막으로 도구를 사용한 시점 |
| 도구별 input/output | JSON 형태 전체 결과 내보내기 가능 |

### 10-4. 현재 구현 상태

| 항목 | 상태 |
|------|------|
| portfolio_entries 저장 | 구현 완료 |
